# Development Workflow

## Purpose

This document defines the engineering workflow followed throughout the project.

The objective is to ensure that every feature is designed, implemented, tested, and documented consistently.

---

---

# Feature Development Process

1. Define the problem.
2. Identify functional requirements.
3. Identify non-functional requirements.
4. Write the feature specification.
5. Review the specification.
6. Produce an implementation plan.
7. Implement in small tasks.
8. Test the implementation.
9. Update documentation.
10. Commit changes.

# Claude Code Usage

Claude Code is used as an engineering assistant.

Claude should:

- Read relevant documentation.
- Summarize understanding.
- Produce implementation plans.
- Implement approved tasks.
- Review completed work.
- Suggest improvements.

Claude should not invent requirements or architecture.

---

# Session Workflow

Every development session begins with:

1. Define today's objective.
2. Identify documents to review.
3. Confirm expected deliverables.
4. Define constraints.

Every session ends with:

- Review
- Testing
- Documentation updates
- Git commit
